# Rule — UI guidelines

1. Responsive: mobile-first; breakpoints 640 / 768 / 1024 / 1280
2. Data tables with >20 rows must have pagination + sticky header
3. Every destructive action (delete, cancel order) → confirm dialog, red danger button placed on the right
4. Loading: skeleton for pages, spinner for buttons; never leave a blank screen
5. Empty states must guide the next action; never show a bare empty table
6. Forms: validate inline on blur, error message below the field, clear Vietnamese wording
7. A11y: every input has a label, contrast ≥ 4.5:1, keyboard operable
8. Toasts auto-dismiss after 3s, at most 1 toast at a time
